## Test Set: GSO-30

**30 objects**, canonical SyncDreamer eval protocol.

### Front-View Images

`data/gso30_renders/{obj}/rgb_0.png` — pre-verified 256×256 front views copied
from `data/gso_fronts/{obj}.png` (1024×1024 RGBA originals, composited onto
white, resized with LANCZOS).

Source `gso_fronts/` images are **pre-verified** canonical fronts from a prior
project; not generated programmatically. This avoids the per-object rotation
derivation complexity (IPS3D `get_angle_deg()` + Blender conventions) that was
explored but ultimately unnecessary given the availability of these reference
images.

**Note:** `gso30_renders/` contains only `rgb_0.png` per object (the single
front view needed for Wonder3D input and VLM captioning). The 6-view renders
used previously were from an intermediate debugging phase and are not retained.

### Novel-View Evaluation Renders

`data/gso30_eval_novelviews/{obj}/view_{00..15}.png` — 16 GT novel views at
22.5° azimuth spacing, rendered via IPS3D `blender_script_v2.py`. Used for
PSNR/SSIM/LPIPS evaluation of CaptionHead readout vs Wonder3D baseline.

## Rendering Pipeline Notes

### Objaverse GLB Hierarchy Bug (fixed in job 71822)

Objaverse/Sketchfab GLBs typically use a root-Empty → child-Empty → Mesh
hierarchy. Prior rendering applied `set_location/set_scale` to leaf Mesh
objects, which only changed LOCAL transforms — the world-space position was
controlled by the parent Empty. Fix: normalise by applying transforms to root
objects (returned by `_load_glb()`), compute bounds from mesh children only,
and call `bpy.context.view_layer.update()` to flush the dependency graph.

See `scripts/blender_render_6views.py` and git history for implementation.

### GSO OBJ Rotation (for reference, not used in final pipeline)

GSO OBJ files use Y-up convention. The rotation sequence to align a GSO
object's canonical front with the camera at azimuth=0 is:

```
rotate_x = -90           (Y-up -> Z-up)
rotate_z = -base_angle   (from IPS3D get_angle_deg())
```

Where `base_angle` is the per-object value from IPS3D. This is implemented in
`scripts/render_gso30.py` + `configs/gso30_per_object_angles.yaml` but
superseded by the `gso_fronts/` copy approach for final data preparation.
